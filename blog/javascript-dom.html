JS Fundamentals

JavaScript and its relationship to HTML and CSS

It’s week 3 of the DevAcademy foundations course and we’ve taken a dive into the world of JavaScript. In the following paragraphs I’m going to explain what JavaScript is, and the fundamentals we’ve learnt so far.

JavaScript is a programming language that works in relationship with HTML and CSS. To explain this relationship I’m going to use the analogy of a Bicycle.

In order for a Bicycle to be built there needs to be a blueprint showing the parts list and the structure of how these parts are put together (HTML). These parts aren’t all the same! They each have their own unique attributes that allow them to do a specific job or have a particular aesthetic (CSS). These attributes could be the paint colour, the parts material or it’s size. Once we’ve assembled the bike, there has to be a form of interaction between these parts for the bike to move (JavaScript).

As illustrated above, HTML provides the elements of a website and the structure of the page. CSS gives unique attributes and styling to each of these elements. JavaScript determines how these elements interact with each other and what causes them to interact (the trigger).

Now that we have an illustration of Javascipts relationship with HTML and CSS I’ll go further into the details of the Javascript fundamentals we’ve learnt this week.

Control flow and loops

Imagine you’re going to the supermarket to buy groceries for cooking a late night cook up. There’s several steps that need to occur for this to happen:

  1. Write a shopping list.
  2. Get in the car.
  3. Drive to the supermarket.
  4. Walk through the supermarket picking up the needed ingredients.
  5. Go to the check-out and purchase the ingredients.
  6. Drive home.
  7. Get cooking.

These steps can’t be completed out of order. We can’t check-out before selecting our products and we can’t make our meal before driving home. There’s a set order for these steps to be followed and if this order isn’t followed we won’t be able to make our glorious meal.

This order is called the control flow. When Javascript files are read they’re read from the first line to the last line in sequential order. There are however, some structures that change the control flow. Loops are a structure that changes the control flow. Lets continue using the example of a supermarket to illustrate loops.

When you arrive at the checkout the cashier performs a loop. They:

  1. Grab the nearest product on the conveyor belt.
  2. Scan the product.
  3. Place the product in a paper bag.

And they continue to do this loop until they reach the total number of products selected. This loop would be presented in code like:

for(i = 0; i < productOrder.length; i++)
{ grab product
scan product
place product in bag}

Once the loop is completed, the cashier is able to move on to the next steps of their code:

  1. Saying would you like a receipt? (Good example of a boolean)
  2. Saying thank-you, have a nice day.

The DOM

HTML, CSS and JS are held together by the Document Object Model (DOM). The DOM creates a tree view of the objects that represent our entire html document. These objects are known as nodes and can be anything from a div or head tag through to CSS .classes and #IDs. As we know, JavaScript is an object based language and these objects in the node tree are exactly that, JavaScript objects! The DOM is in fact an exact replica of our HTML document but in JavaScript.

Because the DOM gives us access to every part of our HTML document through the use of JS objects, we have the ability to create, change or remove objects or properties, pretty awesome! So if we wanted to add a new property to a JS object we could do this through finding the object with a query and then manipulating it by assigning a new value.

Functions

Functions are made up of multiple lines of code and are created to perform a particular task. They are triggered either automatically (on page load) or through an event (such as a button click).

Functions have three key areas to their syntax/ structure.

  1. The first line uses the function keyword followed by a unique name and then parameters (if applicable) within round brackets.
  2. Within the curly brackets the actionable lines of code are written (what the function actually does).
  3. Outside of the curly brackets the function is called using it’s name and the standard brackets. Within these brackets arguments can be written, these arguments are passed into the function.

Functions are helpful because they are repeatable, meaning you can use a function multiple times throughout a project without having to rewrite the code. The outcome of a function can change depending on the arguments written within the brackets when the function is called. This means that there can be multiple different outcomes of the same function, giving them even more flexibility and repeatability.
